# Git & GitHub Mastery: GitKraken Hands-on Lab

This project is a dedicated sandbox for practicing Git workflows, GitHub collaboration, and mastering version control visualization using **GitKraken**.

## 🎯 Learning Objectives
The goal of this project is to bridge the gap between command-line concepts and visual management, focusing on:
*   Efficient repository management.
*   Visualizing complex branching and merging.
*   Resolving conflicts using GUI-based tools.
*   Synchronizing local and remote environments.

---

## 🛠 Tech Stack
*   **Version Control:** Git
*   **Remote Hosting:** GitHub
*   **GUI Client:** [GitKraken](https://www.gitkraken.com/)

---

## 🏁 Workflow Tasks

### 1. Repository Initialization
*   Create a repository on GitHub.
*   Clone the repository locally using the GitKraken "Clone" interface.

### 2. The Daily Cycle (Commit & Push)
*   **Stage:** Selecting specific changes to include in the snapshot.
*   **Commit:** Writing concise, professional commit messages.
*   **Push:** Uploading local history to the GitHub cloud.

### 3. Branching Strategy
*   Creating feature branches (e.g., `feature/ui-update`).
*   Switching between branches (Checkout) and observing the file system changes.
*   Tracking the "Head" pointer visually on the GitKraken graph.

### 4. Merging & Conflict Resolution
*   Merging completed features back into the `main` branch.
*   Simulating and resolving **Merge Conflicts** using the GitKraken Merge Tool.
*   Using **Pull Requests** (PRs) via the GitHub integration.

---

## 💡 Best Practices Captured
> **Pro Tip:** Use the "Undo" button in GitKraken if you make a mistake with a local commit—it’s a lifesaver for beginners!

*   **Atomic Commits:** Keep commits small and focused on one task.
*   **Sync Regularly:** Always `Fetch` and `Pull` before starting new work to stay updated with the remote.
*   **Branch Naming:** Use clear prefixes like `fix/`, `feature/`, or `docs/`.
